🔧 Splunk & FortiNet 연동 설정 가이드

단계별 상세 설정 방법 - 5분 만에 완료하기

🎯 1단계: Splunk 설정

1

HEC (HTTP Event Collector) 활성화

Splunk Web UI에서 데이터 수집을 위한 HEC를 설정합니다.

📍 접속 경로

Settings → Data Inputs → HTTP Event Collector → New Token
Splunk HEC 설정 화면
2

토큰 생성 및 설정

FortiManager 데이터 수집을 위한 전용 토큰을 생성합니다.
Name: FortiManager-PolicyLookup Source Type: fortimanager_policy Index: fortimanager Enable Indexer Acknowledgment: Yes
⚠️ 주의사항: 생성된 토큰 값은 안전한 곳에 보관하세요. 재확인이 불가능합니다.
3

Index 생성

FortiManager 데이터를 저장할 전용 인덱스를 생성합니다.
# CLI 명령어 ./splunk add index fortimanager # 또는 Web UI에서: # Settings → Indexes → New Index # Index Name: fortimanager # Max Size: 10GB (권장) # Max Buckets: Hot=10, Warm=20
4

대시보드 XML 설치

제공된 XML 코드를 사용하여 대시보드를 생성합니다.

📍 접속 경로

Settings → User Interface → Views → New View → Advanced XML
✅ 완료 후: Apps → Search & Reporting에서 대시보드 확인 가능

🛡️ 2단계: FortiNet 설정

1

FortiManager API 활성화

Policy Lookup 기능을 사용하기 위해 FortiManager에서 API 접근을 활성화합니다.
# FortiManager CLI에서 실행 config system admin setting set gui-curl-timeout 120 set gui-polling-interval 10 set show_add_multiple 1 end # JSON API 활성화 config system global set fmg-status enable end
📝 참고: FortiManager 6.0 이상에서 JSON API가 기본 지원됩니다.
2

Policy Lookup 설정

각 방화벽에서 Policy Lookup 기능을 활성화합니다.
# FortiGate CLI에서 실행 config system settings set gui-policy-lookup enable end # 또는 GUI에서: # System → Feature Visibility → Policy Lookup 체크
3

로깅 설정

보안 이벤트와 트래픽 로그를 FortiManager로 전송하도록 설정합니다.
config log fortianalyzer setting set status enable set server "192.168.1.100" # FortiManager IP set source-ip "192.168.1.10" set upload-option realtime end config log setting set fwpolicy-implicit-log enable set local-in-allow enable set local-in-deny-broadcast enable end
4

데이터 전송 스크립트

FortiManager에서 Splunk로 데이터를 전송하는 Python 스크립트를 설정합니다.
#!/usr/bin/env python3 import requests import json import time # Splunk HEC 설정 SPLUNK_HEC_URL = "https://your-splunk-server:8088/services/collector/event" SPLUNK_TOKEN = "YOUR_HEC_TOKEN_HERE" # FortiManager 설정 FMG_URL = "https://your-fortimanager:443/jsonrpc" FMG_USER = "admin" FMG_PASS = "password" def send_to_splunk(event_data): headers = { 'Authorization': f'Splunk {SPLUNK_TOKEN}', 'Content-Type': 'application/json' } payload = { "time": int(time.time()), "source": "FortiManager", "sourcetype": "fortimanager_policy", "event": event_data } response = requests.post(SPLUNK_HEC_URL, headers=headers, data=json.dumps(payload), verify=False) return response.status_code == 200 # 사용 예시 if __name__ == "__main__": sample_event = { "device": "PERIMETER-FW-001", "src_ip": "192.168.1.100", "dst_ip": "10.0.50.200", "service": "HTTPS", "policy_id": 1842, "action": "ALLOW", "lookup_time_ms": 45 } if send_to_splunk(sample_event): print("✅ 이벤트 전송 성공!") else: print("❌ 이벤트 전송 실패!")

✅ 3단계: 테스트 및 검증

1

연결 테스트

설정된 연동이 정상적으로 작동하는지 테스트합니다.
# HEC 연결 테스트 curl -k https://your-splunk-server:8088/services/collector/event \ -H "Authorization: Splunk YOUR_TOKEN" \ -d '{"event": {"test": "connection", "timestamp": "'$(date)'"}, "sourcetype": "fortimanager_policy"}' # 응답 예시: # {"text":"Success","code":0}
2

데이터 확인

Splunk에서 수집된 데이터를 검색하여 확인합니다.
# Splunk Search 명령어 index=fortimanager sourcetype=fortimanager_policy | head 10 | table _time, device, src_ip, dst_ip, action # 최근 1시간 Policy Lookup 통계 index=fortimanager sourcetype=policy_lookup | timechart span=5m count by device_name
3

대시보드 확인

생성된 대시보드가 정상적으로 작동하는지 확인합니다.
✅ 확인 사항:
  • Device Health Status 패널에 데이터 표시
  • Policy Lookup 차트에 실시간 업데이트
  • Security Events 테이블에 로그 표시
  • 새로고침 시 데이터 자동 업데이트

🔧 4단계: 문제 해결

1

데이터가 수집되지 않는 경우

가장 흔한 문제들과 해결 방법을 확인하세요.
# 1. HEC 상태 확인 ./splunk list httpinput # 2. 인덱스 상태 확인 ./splunk list index # 3. 방화벽 규칙 확인 (8088 포트) iptables -L | grep 8088 # 4. SSL 인증서 문제 해결 curl -k -v https://splunk-server:8088/services/collector/event
2

대시보드가 로드되지 않는 경우

XML 구문 오류나 권한 문제를 해결합니다.
⚠️ 일반적인 문제:
  • XML 구문 오류 (태그 불일치, 특수문자)
  • sourcetype 불일치
  • 인덱스 권한 문제
  • 앱 권한 설정
3

성능 최적화

대량의 데이터 처리를 위한 성능 최적화 방법입니다.
# indexes.conf 최적화 [fortimanager] maxTotalDataSizeMB = 10000 maxDataSize = auto_high_volume maxHotBuckets = 10 maxWarmDBCount = 20 # props.conf 설정 [fortimanager_policy] SHOULD_LINEMERGE = false KV_MODE = json category = Security TRUNCATE = 0
